# FBTI 原创角色视觉设定表

更新时间：2026-06-01

## 目标

这套设定用于把当前 `FBTI web quiz` 的 27 个"看球搭子人格"统一转成原创角色资产，风格参考 `sbti` 的统一角色逻辑，但不复用其具体人物或画面。

适用范围：

- 结果页透明背景立绘
- 单角色结果分享卡
- 首页群像
- 社媒分享图二次排版
- 后续卡牌、贴纸、表情包延展

---

## 总体视觉语言

### 1. 角色骨架

- 统一为低多边形 / faceted low-poly 3D caricature 风格
- 足球梗人物头像或半身 / 全身角色，比例夸张但轮廓清晰
- 保留真实人物或事件的核心辨识点，但整体做成玩具感、游戏头像感的原创卡通化形象
- 色彩干净，块面明确，服装和道具服务于人物梗，不要做写实照片

### 2. 输出规格

- 主资产：透明背景 PNG（必须输出带 alpha 通道）
- 推荐尺寸：`1024x1024` 或 `1024x1536`
- 角色居中，姿态有辨识度，画面留出适当边距
- 不能有场景背景、渐变背景、阴影底板、文字、字幕、水印、logo 字样

补充说明：

- 如果资产用于"结果分享卡"，则允许把标题文案直接做进图片
- 如果资产用于网页结果页嵌入，则优先使用无标题透明立绘版本

### 3. 共用服装规则

- 不使用真实俱乐部或国家队球衣（配色可暗示，但不能出现真实队徽或文字）
- 统一使用"城市休闲 + 看球现场感"的原创服装语言
- 上装可在 T 恤、卫衣、外套、球迷围巾之间变化
- 下装以短裤、长裤、运动裤为主
- 鞋子统一是简化运动鞋，避免细节过多

### 4. 表情规则

- 眉毛是第一情绪器官
- 眼睛只保留黑点或小椭圆，不做复杂高光
- 嘴型控制在 6 类：平、抿、张口、斜撇、坏笑、惊呼
- 同一组内角色情绪强弱可以不同，但不能跳出整套审美

### 5. 道具规则

- 每个角色最多 `1` 个主道具 + `1` 个辅助符号
- 道具优先服务人格识别，不追求真实材质
- 道具形体尽量大块面，适合低多边形语言

### 6. 色彩规则

- 每组先有组色，再做角色微调
- 整套避免颜色过脏或过荧光
- 主色不超过 `2` 个，辅色不超过 `2` 个
- 皮肤、头发、鞋底等基础色尽量共用，减少资产风格漂移

### 7. 当前验证通过的卡片模式

这一版已经验证过一套可用的"单角色结果卡"结构，后续 27 张卡片建议统一按这个模板生成。

版式顺序：

1. 顶部说明句：`你的看球搭子类型是`
2. 中文人格名：例如 `狂人`
3. 英文 code：例如 `MOU`
4. 角色主体：全身原创低多边形角色
5. 底部只保留轻阴影，不额外加说明文字

文案层级要求：

- 第一行最小，作用是解释卡片语义
- 第二行最大，是主标题
- 第三行次于中文名，但仍需明显，作为识别 code
- 不额外加入组名、tag、短描述，避免卡面信息过满

背景要求：

- 统一暖白或米白背景
- 不加复杂纹理，不加大面积渐变
- 不加边框装饰，不做九宫格或拼贴式排版

角色摆位要求：

- 角色始终居中
- 角色完整露出全身
- 角色高度约占卡面可视区的 `58%` 到 `68%`
- 标题区和角色区之间留出稳定留白

色彩要求：

- 相比早期样张，单卡阶段要进一步压缩色数
- 每个角色采用 `1` 个主色家族 + 黑/深灰 + 骨白
- 只有在识别需要时，允许加入 `1` 个小面积强调色

适用场景：

- 社媒分享图
- 测试结果长图首卡
- 小红书/朋友圈/群聊传播图
- 后续导出 PNG 卡面

不适用场景：

- 网页结果页中间的角色插图位
- 需要透明背景的前端嵌入图

当前已验证的试点角色卡包括：

- `狂人 / MOU`
- `詹指 / DAT-A`
- `第12人 / CN-12`
- `内马尔 / NEY-MAR`
- `卡牌大师 / CARD-MA`
- `齐祖 / ZI-DANE`

命名建议：

- 单角色分享卡：`mou-card-v1.png`
- 带中文名和英文 code 的确认版：`mou-card-v2-cn-en.png`
- 透明立绘版：`mou.png`

## 分组配色

| 组别 | 角色气质 | 主色建议 | 辅色建议 |
| --- | --- | --- | --- |
| 看球反应组 | 情绪即时、动作大、现场感强 | 番茄红、亮橙、运动蓝 | 米白、炭灰 |
| 赛后话题组 | 脑力型、输出型、信息型 | 墨绿、书页白、深海蓝 | 黄铜、灰褐 |
| 搭子风格组 | 社交型、群聊型、人格互动强 | 湖蓝、酸橙绿、暖黄 | 奶油白、石墨灰 |
| 足球哲学组 | 观点型、原则型、审美或结果导向 | 酒红、藏青、古铜、米卡其 | 象牙白、深灰 |
| 特殊人格 | 传奇感、戏剧感、异常值 | 紫灰、金色、黑曜色、电青 | 冰白、烟灰 |

## 单角色结果卡模板

这一节用于后续批量生成 27 张结果卡。

### 卡面结构

- 顶部留白
- 一行说明句：`你的看球搭子类型是`
- 一行中文人格名
- 一行英文 code
- 角色主体
- 底部轻阴影

### 统一文案模板

对于任意角色，文案固定为：

```text
你的看球搭子类型是
<中文名>
<英文 code>
```

示例：

```text
你的看球搭子类型是
狂人
MOU
```

### 统一排版要求

- 文字全部居中
- 中文名必须比英文 code 更大
- 英文 code 必须完整保留原始大小写
- 不在卡面加入额外说明句、组名或人格描述
- 角色与文字之间至少保留一个清晰呼吸区

### 统一风格要求

- 背景干净克制
- 角色造型可以天马行空，但版式必须稳定
- 画面重点永远是"人格标题 + 单一角色"
- 禁止回到多人拼板模式作为结果卡终版

当前已验证的试点角色卡包括：

- `狂人 / MOU`
- `詹指 / DAT-A`
- `第12人 / CN-12`
- `内马尔 / NEY-MAR`
- `卡牌大师 / CARD-MA`
- `齐祖 / ZI-DANE`

命名建议：

- 单角色分享卡：`mou-card-v1.png`
- 带中文名和英文 code 的确认版：`mou-card-v2-cn-en.png`
- 透明立绘版：`mou.png`

### 名言使用规则

- 名言优先用于分享卡版本，不必直接做进透明立绘
- 名言应作为角色的次级识别信息，不能压过中文名与英文 code
- 名言默认置于角色下方或卡面底部区域
- 如果名言较长，优先缩小字号，不拆散角色主体的视觉重心

## 全局生成风格要求

> 以下为统一的 AI 图像生成约束，与 `web-quiz/ref/image_generation_prompt.md` 保持一致。

- 低多边形 / faceted low-poly 3D caricature 风格
- 足球梗人物头像或半身 / 全身角色，比例夸张但轮廓清晰
- 角色居中，姿态有辨识度，画面留出适当边距
- 透明背景，必须输出带 alpha 通道的 PNG
- 不能有场景背景、渐变背景、阴影底板、文字、字幕、水印、logo 字样
- 保留真实人物或事件的核心辨识点，但整体做成玩具感、游戏头像感的原创卡通化形象
- 色彩干净，块面明确，服装和道具服务于人物梗，不要做写实照片

推荐尺寸：`1024x1024` 或 `1024x1536`

### Negative Prompt

```
photorealistic, real photo, realistic skin pores, background, stadium background, crowd background, text, caption, watermark, brand logo text, messy details, blurry, cropped head, cut off feet, flat 2D vector, anime style, oil painting, horror, grotesque, duplicate body, extra fingers, distorted face, low resolution, opaque background, white background, checkerboard background rendered into image
```

## 执行标准

出图时统一检查这 5 条：

- 不像真实球员，不落回"球星替身"路线
- 不靠标题文字也能认出人格方向
- 同组角色放在一起时，配色和面部语言有亲缘性
- 单个角色缩小到结果页尺寸时仍可辨认主道具
- 放进当前结果页卡片后，不会因为留白、姿态或重心失衡而显得飘

这份设定表的目的不是一次定死所有细节，而是先把"原创人格角色系统"从散点想法，收束成能批量生产的统一规范。
